# Security

This is a personal Claude Code configuration repo, not a production
library. Security expectations are limited and match the rest of the
repo: no support, no SLA, no commitment to respond.

## Reporting

- **Low-severity findings** — config gotchas, doc errors, broken
  examples, hooks that misbehave: open a public GitHub issue.
- **Credential exposure or anything actively harmful** — keys / tokens
  / personal data committed to history, an example that leaks secrets,
  a hook or skill that could damage a user's environment: use GitHub's
  private vulnerability reporting (repo **Settings → Security →
  "Privately report a vulnerability"**). If that's not available to
  you, open a minimal public issue requesting a private channel rather
  than disclosing details publicly.

No SLA on response times. Same expectations as the rest of the repo —
this is a personal configuration that happens to be public, not a
maintained product.
